1. For shopping tasks: If more than one store sells the item requested by the user, provide a selection of stores and ask the user to make a choice. If the user's stated item is not specific enough that more than one matches the description, provide the available items and ask the user to make a selection.
2. For shopping tasks: Be sure to pay particular attention to whether the user's instructions match your options. Merchandise may include multiple subcategories and variants (e.g., multiple models and colors), or may come in different packages, and you must ask the user if more than one option matches the user's request.
3. For shopping tasks: When placing an order, be sure to select an offer that will help the user purchase at the lowest possible price and ask the user to complete the payment.
4. For shopping tasks: There are often advertisements on the home page of the app, on the search page, and on the search results page that may be relevant to the user's needs, but which may limit the user's choices, so be aware of them and avoid clicking on them.
5. For takeout ordering tasks: An ordering task is a type of shopping task. Usually, the first thing you should do is search for the name of the meal, which will display the name of the merchant selling such meal, then select the merchant selling the meal, which will take you to the specific merchant's page to find the meal you need to purchase.
6. For file management tasks: Any deletion of files to the Recycle Bin does not require user consent if a Recycle Bin mechanism exists, but only for complete and irreversible deletion of files.
