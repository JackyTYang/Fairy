index: the index position of the current node in its parent node (integer)
text: the text content displayed in the control, which may be empty
resource-id: the resource ID of the control, often used for UI automation positioning
class: the class name of the control (full Java class name), indicating the control type, such as TextView, ImageView
package: the application package name to which the control belongs
content-desc: content description, usually used for barrier-free services or test assistance (such as "map bubble button")
checkable: Boolean value, whether the control supports checking (such as CheckBox)
checked: Boolean value, whether the control is currently checked
clickable: Boolean value, whether the control can be clicked
enabled: Boolean value, whether the control is in an enabled state
focusable: Boolean value, whether the control can get focus
focused: Boolean value, whether the control currently gets focus
scrollable: Boolean value, whether the control is scrollable
long-clickable: Boolean value, whether the control supports long-press operation
password: Boolean value, whether the control is a password input type (the content will be hidden)
selected: Boolean value, whether the control is in a selected state (used for lists, options, etc.)
bounds: The position boundary of the control on the screen, the format is [x1,y1][x2,y2]
rotation: The rotation angle of the control (integer, usually 0)
NAF: Non-standard attribute, meaning Not Accessibility Friendly, which means that the control may lack an accessibility label
